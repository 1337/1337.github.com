Gulp Adventures

I don’t actually know much about JavaScript development when it comes to npm, grunt, gulp, and volo. They all seemed like… things, to me. Do you have a webapp that handles your models, controllers, and views, but somehow you need static files to be generated using these new workflows, but have no idea how? That’s what I am going through! Here outlines my learnings.

Install (almost) everything with --save

Unlike apt-get, nothing you save with npm and bower will be available to your environment by default. To persist these packages across multiple developer machines, run npm install and bower install with the --save flag, which records the packages you installed into package.json and bower.json, respectively.

Always do the hippest thing

Java has a reputation for having robust, tried-and-true methods for doing everything. Want to run a webapp? Install spring’s jars. Want to install a package? You will probably be using maven. However, in JavaScript development, everything is making a new flavour of everything, where no standard is made and/or obeyed. Ignoring the fact that no one has any idea what is going on, it is also apparent that the only way to stay afloat in the web development community is to use the most recent builds of everything, in order to shorten your dependency tree (example: if you need to use an older version of Backbone, then all other dependencies will install newer versions of Backbone alongside yours).

Never settle for Grunt, because Gulp exists. If Gulp has a newer alternative, then switch to it immediately. There is no such thing as resilience in this field – the only thing you need to do is to accept change.

Writing Annoying Blog Post Titles

I am a contributor of Downworthy, a Chrome extension that removes annoying words from online headings and bodies, sometimes known as linkbaits or clickbaits. Since it is disappointing to click on an interesting-sounding link, for example:

How I earned a million dollars overnight

Just to be followed by a slow page, megabytes of useless JavaScript and images, and a ton of ads:

I bought the lottery and won. Here are five paragraphs about the process, how much I spent on the ticket, how I felt, and how it affects you and me. EXCLUSIVE: 35 SHOCKING SELFIES I TOOK WHEN I COLLECTED MAH MONEY

Naturally, to avoid future incidences of clicking on these mid-grade links, we must first analyse the components of a linkbait title, and how we can neutralise such titles so that normal titles, which may well contain more interesting stories, can get their fair share of our clicks.

Anatomy of a stupid linkbait title

Let’s first look at the front page of Buzzfeed, one of the most pathetic websites on the planet.

You will find their titles fit in one or more of these three categories.

Lists, particularly lists that somehow you can relate:

  • 17 Things To Do When You Are Bored Out Of Your Mind
  • 5 Brilliant BuzzFeed Newsletters That Will Always Brighten Your Day
  • 19 Superfood Combinations That Will Make You Feel Like A Champion
  • 21 Embarrassing Things Every Muggle Secretly Does
  • 19 Signs You’re Definitely Eric Forman From “That ’70s Show”

Titles beginning with Who/What/When/Where/Why, whether or not they are actual questions:

  • What Type Of Shark Are You?
  • Which Sci-Fi Hero Should You Hook Up With?
  • How One Company Is Building An Internet Connection Through The Arctic Thanks To Climate Change
  • What It’s Like To Go Blind
  • What The World Was Like In 1974 When Richard Nixon Resigned

Exaggeration:

  • This Fact About Killer Whales Will Blow Your Mind
  • The Most Fab And Drab Celebrity Outfits Of The Week
  • 24 Pictures That Prove That Life Hacks Have Gone Way Too Far
  • This Timelapse Video Of Pyongyang, North Korea Is Beautiful And Intriguing (it wasn’t)

Reading comprehension:

  • Most viral articles by Buzzfeed have vocabulary aimed at 10-year-olds.

Takeaways

There are only two things we have learned from this exercise:

  • Annoying titles overpromise
  • Annoying articles underdeliver

To avoid wasting time on these posts (including this one, for some), I suggest installing the downworthy plugin!

Creating a Blog

Heard from someone I know (but wish not to disclose his/her identity) that Hexo is a nice framework to use for generating a static blog. Having never done that before, I thought, what the hell!

My experience with Hexo has been very pleasant (or… much better than my previous encounters with node and npm). Here are a few take-aways:

  • Incorrect permissions of the local npm cache (caused by sudo) can and will fail most of the npm- and nodejs-related tasks that you will be running.
  • Inserting images in Hexo (apparently /source is the root path, not / as the config suggests)
  • livereload requires a browser plugin.
  • You need to click on livereload’s button in the browser…
  • EJS, because the theme uses it
  • Stylus, because that’s also what the theme requires; it allows semicolons and brackets to be omitted anywhere you want!
  • Apparently SASS is deprecating its bracket-free syntax
  • How to create referenced links.
  • How to deploy this blog. Booboo! You don’t commit and push – you deploy and push to a draft branch.
  • Deploying blogs to github under a custom domain.

It has been an amazing learning experience! I’m taking a lot of notes as I go. More to come tomorrow.
