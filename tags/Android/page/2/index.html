Utilising Android System Memory With Sysctl and Minfree

Article Pre-requisites

  • A rooted device with debug enabled
  • Linux knowledge
  • ADB server
  • Read/write permissions on /
  • A working recovery partition

What’s happening?

If you have a mid-range Android device (hereafter referred to as a “phone”, whether or not yours is a phone), chances are that you have opened one app, opened another app, then switched back to the first app, just to see that the first app had been killed, and you had to start it again. “Oh,” you thought to yourself, “maybe my phone is just old. Maybe it is running out of memory,” but you would (often) be wrong. Tuning Minfree/LMK

Minfree is Android’s own implementation of user-level memory management, on top of the kernel’s kernel-level memory management. It is also known as the LowMemoryKiller (LMK). It is responsible for the apps being killed in a timely manner. The parameters are as follows:

  • FOREGROUND_APP: The active app
  • VISIBLE_APP: Usually an app that is not the active one, but still visible (e.g. with an active notification, or some overlay thing you are running)
  • SECONDARY_SERVER: Like your launcher.
  • HIDDEN_APP: Idle apps.
  • CONTENT_PROVIDER: Services that provide content, e.g. contacts and calendar services.
  • EMPTY_APP: Apps not doing any work.

Often, your phone ships with ridiculously high values like these:

(Divide each number by 256 to get the number of MB for each category)

The values above tell Android to start killing apps when there are 318MB of free memory, and kill visible apps when there are 90MB left. This causes the regular “my app is killed” problem. They are set so high because it is nicer for the manufacturer to kill the apps for the user, than it is to have them reboot their devices when an OOM (out of memory) event occurs. Higher minfree values also allow manufacturers to report (slightly) higher battery life when tests are conducted in a laboratory setting.

FAQ: Does keeping apps in memory increase battery usage?

Yes. Any app kept running, even in the background, will cost CPU cycles. Be careful what you wish for. Consider using Greenify to make sure certain apps (Facebook, I am looking at you) never run in the background.
On the plus side, if you don’t use your phone a lot, this makes your user experience vastly better.

FAQ: Should I wipe my apps out of the recent apps screen?

Keeping them there will not hurt.

FAQ: Will tuning my memory impact my read/write speed?

Yes. Occasionally, there may be a positive effect.

Tuning background apps limit (Oneplus 3 Only)

An article claims that, by editing ro.sys.fw.bg_apps_limit in the device’s build.prop file from 20 to 40, more apps will be kept in memory for longer.

Setting values permanently

Knowing minfree values are reset on reboot (sometimes reset immediately after setting, as well), you might want to apply the values every reboot. You have three choices:

  1. Use a kernel manager to set values for you on reboot. Some good ones: Trickster MOD, Kernel Adiutor, Synapse,
  2. Create an executable /etc/init.d/123_script_file to apply the values for you, or
  3. Flash the V6 SuperCharger Script through the device’s recovery.

FAQ: Should I download a memory booster?

No.

Nobody Cares About Your Stupid App

As a mobile developer (if I can be called as such), I download competitor apps for business intelligence. This weekend, I downloaded Didi Kuaidi, whose company has exceeded Uber China’s market cap threefold ($25B vs $8B).

With this sort of market cap, they must have hired hundreds of thousands of smart Chinese developers to make a killer Android app, right? As it turns out, any question I ask here can be answered with ‘no.’

First thing after opening the app–which required more than a page of permissions to install–you are greeted with “Do you want to update?”

A stroke of genius from them: since Google Play Services is not that common in China, people there turn on “allow installations from unknown sources”, which allows/forces apps to update themselves like this.

After you update, you get this glorious map of sooty Beijing:


Why the map centers at Beijing, it is simple: the company is headquartered there. Why, then, can a startup 1/1000 of that size not have to have its app simply center its map in Toronto by default, is mind-boggling.
The app also does not try to obtain the user’s location, which is presumably where most trips would start.

I have received many, many complaints about the map (in my app, whichever one you think it is) being slow, and we just need to fix, optimise, and make it pretty. Well, look at this $25B map:

The map is some sort of custom Google Maps (or a very close imitation of it), which highlights the regions in which Didi Kuaidi offers service. I like that. Instead of doing just that, I have decided to disappoint my users after they perform a search. This allows me to generate a “demand heat map” with these failed searches.

There are many cute intro screens when you try to access the “UberPool of China”. I like those too.


Just when you think you are about done looking through this app, and would like to check the ‘Activity Centre’ to see what’s new. And lo, in all its glory:

Nobody cares about your stupid app. Nobody cares if it is polished. All it needs to do is work.

Making My First App

With Android being all the hype these days, I have decided to make an app.

Having abandoned Java since 2010 (excluding that one bug fix in one of the company’s repositories in 2014), I really had nothing to lose. The syntax, the types, the way projects are built, event handling… I have forgotten all that. This app will be a fresh start to programming something again – something that isn’t run in the browser.

Downloading the SDK

Have you used eclipse? Of course you have. But do you like Eclipse? That’s a much less certain question to answer.

It was actually JetBrains’ release of the first usable SDK that had my interest. When version 0.8 was released, all the screenshots looked like it looked like PyCharm, which is what I use for work and personal projects already. Presumably, with Android Studio, I would be able to add code, edit interfaces, and code like a normal human being, right…?

That is correct!

Unlike Eclipse, the de facto Java IDE from the software giant of Ottawa, JetBrains products are not complete and utter shit; instead, they are designed to get out of the way of the developer. Hell, I don’t even know how to change the editor’s background from white to black in Eclipse.

Creating a new project

Creating a new project is easy enough. Android Studio has a nice wizard that guides you through the process. The wizard creates a project, an activity, and a layout / “content view” inside that activity (which is some kind of abstraction… see later posts.)

Creating the UI

I remember from the Visual Basic 6 days that UIs were extremely easy to make and configure. For instance, if you wanted a number box with a default value, min and max, you would drag one of such element into the window (“form”), enter these values on the right hand Properties pane, and call it a day.
