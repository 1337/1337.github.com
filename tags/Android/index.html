How to Sign an APK

  1. First, make a keystore

    keytool -genkey -v -keystore ~/.keystore -alias alias_name -keyalg RSA -keysize 2048 -validity 10000 # or something along those lines

  2. Then, make your unsigned APK

    build (...)

  3. Then, zipalign the APK, which reportedly reduces the app’s memory usage:

    zipalign -v -p 4 my-app-unsigned.apk my-app-unsigned-aligned.apk

  4. Finally, sign your app with that keystore you just made.

    apksigner sign --ks ~/.keystore --out my-app-signed-aligned.apk my-app-unsigned-aligned.apk # apksigner only available starting SDK 23 and up

You should make this a script.

What I Learned Helping an Android Developer Fix a Bug

tl;dr: Not much.

While mucking about with my phone in traditional Chinese, I discovered that some of the words in Sleep as Android show up as simplified Chinese. As that is easily an act of war, apparently, I notified the SaA team of this issue.

The reply I got was an invitation to translate the zh-rTW strings file for the app. Immediately, I learned (1) that Android’s zh strings suffices have an r before it, whatever it is.

I accepted the invitation. Opening the document, I found that all of the strings are, well, already translated. What gives?

Googling the problem, I found that zh-rHK is not even an Android locale, and that its existence in CyanogenMod is purely generosity. Well, or so I think. The problem with doing that is zh-rHK actually falls back to zh-rCN, which is simplified Chinese. Blasphemy!

I suggested the SaA team copy the zh-rTW file as zh-rHK, so as to maintain the sovereignty of the People’s Republic of Hong Kong. They sent back an updated beta and it was fixed. We did it, Hong Kong!

‘Twas a good joke.

Fixing ES File Explorer's Root Explorer

Since ES File Explorer has a root app backup option that I use extensively, I cannot switch file explorers willy-nilly. And if you use SuperSU, then you may notice that the app (I am using a resigned version of the 3.2.x.x app) cannot request root properly. After some research, it turns out the explorer (even the 4.x app) is just crappy, and cannot find the su` binary.

The fix is as follows:

  1. adb shell into your device.
  2. Run a root shell: su
  3. Make the system partition writable: mount -o remount,rw /system
  4. Symlink su to the system partition, where the explorer expects it: ln -s /su/bin/su /system/bin/su
  5. Remount /system as read-only if you’re paranoid about anything.

Tuning Android IO Schedulers

If for whatever reason you feel like you should tune your Android device’s IO scheduler, despite it rarely lags, then this guide is for you. Otherwise, the cheatsheet below might be more appropriate.

This post does not apply to you and/or your device.

First choice: ROW

ROW stands for “READ Over WRITE” which is the main requests dispatch policy of this algorithm. The ROW IO scheduler was developed with the mobile devices needs in mind. In mobile devices we favor user experience upon everything else,thus we want to give READ IO requests as much priority as possible. (…)

I have never had a problem with ROW. If you don’t have ROW available from the kernel, maybe…

Second choice: Deadline

With separate read and write priority queues, Deadline also gives read a higher priority. You might need to tune the behaviour to favour read even more, however.

Default / Last resort: CFQ

Processes can have an IO priority. The scheduler is a bit more for rotating disks than it is for solid state memory.

Last resort: NOOP

Leaves it to the IO controller. It might be beneficial to encrypted devices, but I personally see more lags here and there.
