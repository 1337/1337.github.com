Fixing ES File Explorer's Root Explorer

Since ES File Explorer has a root app backup option that I use extensively, I cannot switch file explorers willy-nilly. And if you use SuperSU, then you may notice that the app (I am using a resigned version of the 3.2.x.x app) cannot request root properly. After some research, it turns out the explorer (even the 4.x app) is just crappy, and cannot find the su` binary.

The fix is as follows:

  1. adb shell into your device.
  2. Run a root shell: su
  3. Make the system partition writable: mount -o remount,rw /system
  4. Symlink su to the system partition, where the explorer expects it: ln -s /su/bin/su /system/bin/su
  5. Remount /system as read-only if youâ€™re paranoid about anything.
