What happens when Apple rejects your app

Or: “How to get your entire stack ready for IPv6 in a Day”

The folk here at Tringa Semiconductors (me) makes an iOS app that converts social shoppers back into browsers.

  • Performance - 2.1

    We discovered one or more bugs in your app when reviewed on iPad running iOS 10.0.2 on Wi-Fi connected to an IPv6 network.
    Specifically, an error occurred when attempting to log in.
    We’ve attached screenshot(s) for your reference.

  • Try to reproduce locally

  • Nothing is wrong locally
  • Read potential reasons
    http://blog.ionic.io/apples-switch-to-ipv6-what-it-means-for-your-app/
  • Realise IPv6 isn’t being used at all
  • Try to set up ipv6 only network
    http://www.internetsociety.org/deploy360/blog/2012/04/where-are-the-ipv6-only-wi-fi-routers-and-access-points/
  • http://unique-local-ipv6.com/
  • Test again http://test-ipv6.com/
    Still doesn’t work
  • https://developers.google.com/speed/public-dns/docs/using
    2001:4860:4860::8888
  • Test API itself http://ready.chair6.net/?url=https%3A%2F%2Fbeta.rideco.com
    Doesn’t work
  • Configure route53 to dualstack
    http://docs.aws.amazon.com/elasticloadbalancing/latest/classic/elb-internet-facing-load-balancers.html#internet-facing-ip-addresses
    Wait TTL
  • Test API again http://ready.chair6.net/?url=https%3A%2F%2Fbeta.rideco.com
    Still doesn’t work
  • Modify apache2 conf
    http://www.cyberciti.biz/faq/ipv6-apache-configuration-tutorial/
  • Hang on, you don’t need to change the apache2 conf because dualstack is ipv4 behind the load balancer
  • wget -6 from another machine
    https://ubuntuforums.org/archive/index.php/t-2048623.html
    wget -6 https://[2406:da00:ff00::3401:a240]/
  • wget says address unreachable
  • ping6 2406:da00:ff00::3401:a240
    ping6 also says network unreachable
  • realise the domain is hosted as a CNAME record
    https://scottlinux.com/2015/03/09/how-to-configure-route53-dns-for-an-elb-in-aws/
  • change to A and AAAA record pointing to dualstack
    doesn’t work either
  • “we might need a new (aws) load balancer in classic mode”
  • Apple might have just made an error
  • Undo everything
  • Apple
